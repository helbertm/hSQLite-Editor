Guia rápido
Use F4 para ocultar/mostrar este painel.
Clique em uma tabela com Cmd/Ctrl para inserir o nome.
Clique em uma coluna para inserir tabela.coluna.

Autocomplete restrito:
from/join/update/into sugere tabelas/views.
tabela., view. ou alias. sugere somente suas colunas.
Ctrl/Cmd+Espaço força a abertura quando houver contexto válido.

Execução:
Consultas pesadas rodam, mantendo a interface responsiva. O botão Cancelar encerra o worker.

Ordenação:
Clique normal cria uma ordenação única.
Shift+clique no cabeçalho adiciona critério, como ORDER BY a DESC, b ASC.

Colunas:
Arraste a borda direita do cabeçalho para aumentar/diminuir. Duplo clique na borda restaura a largura automática.
Ctrl/Cmd+clique no cabeçalho congela da primeira coluna até a coluna clicada.
Arraste o cabeçalho da coluna para mudar sua posição.

    Ajuda Geral do hSQLite Editor
    
      
    Ferramenta para análise de bases SQLite com foco em suporte/TI e uso didático em aulas de banco de dados.
    
      
    
        
    
          
    1. Começando
    
          
    Use Escolher .db para abrir uma base. Depois escreva SQL e execute com F5.
    
          
    Extensões aceitas para bancos SQLite: .db, .sqlite, .sqlite3, .db3, .s3db, .sl3, .sqlite.db.
    
        
    
        
    
          
    2. Área SQL
    
          
    Abas independentes, histórico, favoritos, abrir/salvar arquivo SQL e atalhos para fluxo rápido.
    
        
    
        
    
          
    3. Schema
    
          
    Filtre objetos por nome e por tipo. Clique em colunas para inserir objeto.coluna no editor.
    
        
    
        
    
          
    4. Resultados
    
          
    Ordenação multi-coluna, paginação, seleção de linhas, exportação CSV/JSON e cabeçalho sempre visível.
    
        
    
        
    
          
    5. Persistência
    
          
    Sessão, histórico, favoritos e preferências ficam no navegador (localStorage/IndexedDB).
    
        
    
        
    
          
    6. Modo offline
    
          
    O arquivo oficial deve abrir direto no navegador como um HTML standalone, inclusive em file://.
    
        
    
        
    
          
    7. Reporte de bug
    
          
    Use o botão Reportar Erros para abrir o GitHub Issues do projeto.
